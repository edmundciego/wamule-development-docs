# Screenshot and training-data policy

All documentation screenshots and videos must use test accounts and fictional names, phones, email addresses, lots, payments, receipts, and documents. Add a visible **Training Data** marker to every captured frame.

Do not capture or publish customer names/contact details, signatures, contracts, receipt numbers, bank references, document URLs, access tokens, API keys, Supabase/Resend/Gemini credentials, production settings, or internal audit exports.

Before approval, inspect full-resolution media for exposed values, crop/redact where needed, write useful alt text, and record the source test environment. Do not use real production customer data even if names appear obscured.
