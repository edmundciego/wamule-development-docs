# Documentation style guide

## Voice

Write for Wamule Development staff: clear, calm, specific, and accountable. Use “Wamule Operations Platform” for the software and “Wamule Development” for the business.

## Article structure

- One direct H1.
- Start with purpose and operational scope.
- Use ordered steps for staff action; tables only when comparing several fields or states.
- State role, status, verification, escalation, and completion criteria for workflows.
- Add a warning/caution where an action is privileged, irreversible, financial, advisory, or not automated.

## Accuracy boundaries

Never imply that AI approves applications, confirms a payment, changes a balance, sends email automatically, or makes a human decision. State when a reservation, site visit, deposit status, report, or summary is not an authoritative financial or legal action.

## Inclusive and accessible writing

Use meaningful headings, descriptive links, plain language, and concise alt text. Do not rely on color or image-only instructions. Define a term the first time it appears or link to the glossary.
