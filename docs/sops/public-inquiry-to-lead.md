# SOP — Public inquiry to lead

**Purpose:** create one accountable sales record from a public information inquiry or application.

| Field | Standard |
| --- | --- |
| Trigger | Public inquiry or application is submitted. |
| Responsible role | Sales coordinator or assigned sales staff. |
| Required information | Name, safe contact details, source, lot interest, duplicate context, owner. |
| Steps in platform | Review new item; open/create linked lead as the deployed workflow provides; check duplicate flag; assign owner; create/verify first follow-up. |
| Expected status changes | Lead starts at `new_lead`; application remains `Pending Review` when applicable. |
| Required documents | None unless the public form supplied approved supporting context. |
| Financial checks | None; do not record a deposit or payment at intake. |
| Follow-up deadline | First-contact target approved by Wamule Development. |
| Escalation conditions | Possible duplicate, unavailable lot, missing contact method, or no owner. |
| Completion criteria | Owner, next action, due date, and initial review outcome are recorded. |
| Audit/accountability evidence | Lead activity, task, source link, and supported audit event. |
| Common mistakes | Treating a lead as approved; creating duplicate leads; leaving no next action. |
