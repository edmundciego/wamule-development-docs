# SOP — Reservation and deposit review

**Purpose:** review readiness evidence without bypassing financial verification.

| Field | Standard |
| --- | --- |
| Trigger | Deposit due, proof submitted, overdue, or reservation close to expiry. |
| Responsible role | Finance/Admin with sales coordination. |
| Required information | Reservation, expected amount/due date, proof, linked payment, bank/receipt details. |
| Steps in platform | Open reservation and payment/customer records; verify actual evidence; update readiness; link existing payment only when correct; schedule next action. |
| Expected status changes | Deposit pending/proof submitted/confirmed/overdue/waived/cancelled; reservation status if approved. |
| Required documents | Approved proof, receipt photo, signed note, or bank evidence. |
| Financial checks | Verify ledger transaction, method, amount, bank reference, receipt number, and proof. |
| Follow-up deadline | Before due date/expiry; overdue cases immediately. |
| Escalation conditions | Amount mismatch, no evidence, duplicate claim, conflicting reservation, or unclear bank record. |
| Completion criteria | Readiness decision, financial verification, owner, and next state are documented. |
| Audit/accountability evidence | Reservation activity, payment record, document link, Audit Trail. |
| Common mistakes | Marking confirmed from buyer statement alone; changing balance from reservation page. |
