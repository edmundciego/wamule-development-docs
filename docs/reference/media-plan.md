# Training media plan and privacy policy

Use article media labels to plan text, annotated screenshot, short walkthrough video, workflow diagram, checklist, and fictional scenario assets.

Every future image/video must use a test account, fictional customer/lot/payment data, a visible **Training Data** marker, and redaction of internal identifiers where needed. Do not expose customer contacts, signatures, contracts, receipt numbers, bank references, document URLs, credentials, production configuration, tokens, API keys, or audit exports.

Place approved files in `static/media/` with descriptive names and alt text. Review assets as carefully as written content before publication.
