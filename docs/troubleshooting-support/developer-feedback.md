# Submit developer feedback and support requests

Use the in-app **Send Feedback** control for questions, bugs, data issues, and feature requests.

Include:

- page/feature and safe record reference;
- action attempted;
- expected and actual result;
- whether work is blocked;
- date/time and screenshot if safe;
- priority: Low, Normal, High, or Urgent.

Never include passwords, API keys, bank credentials, tokens, or unredacted customer documents. For suspected unauthorized access or exposed data, use the organization’s approved security escalation channel: **[confirm owner and contact before launch]**.

**Media:** Annotated screenshot, checklist.
