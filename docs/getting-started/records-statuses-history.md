# Records, statuses, dates, and activity history

Each page represents a business record: a lead, lot, application, customer, contract, payment, reservation, request, or task. Update the authoritative record rather than relying on a message or memory.

- A **status** describes the current documented state, not a prediction.
- A **due date** requires a named owner and follow-up.
- An **activity** explains meaningful contact or a change in context.
- The **Audit Trail** supports accountability for supported record changes.
