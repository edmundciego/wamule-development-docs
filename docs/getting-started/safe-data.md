# Safe use of live and training data

Use production records only for authorized business work. Training, screenshots, videos, and demonstrations must use test accounts and fictional names, contact details, lots, payments, and documents.

Never place passwords, access tokens, API keys, customer contact data, signatures, receipt numbers, bank references, document URLs, or confidential audit exports in training material or feedback.

:::danger Do not use destructive tools for routine fixes
The restricted data-management purge workflow is for approved test or incorrect records under Super Admin control. Use normal correction, cancellation, void, archive, or documented escalation procedures for real operational history.
:::
