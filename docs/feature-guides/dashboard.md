# Dashboard

The Dashboard is the daily operational overview. It surfaces follow-ups, upcoming visits, sales movement, reservation/deposit readiness, collections alerts, post-sales work, and lot totals.

Use **Today’s Attention** to identify the actual records that need work. Each listed item should include the affected buyer or customer, action, deadline, owner, status, and an **Open action** link. Use **View all** to open the relevant filtered queue.

Supporting metric cards are actionable when their value is non-zero. For example, overdue follow-ups open the Leads workspace, applications waiting open Applications, collection alerts open Collections, and post-sales blockers open the customer post-sales area.

The Dashboard is live. It uses current source records rather than saved AI wording, so confirm the source record before acting. If a count is non-zero but records cannot be loaded, use the retry action and report the issue rather than relying on the number alone.

See [Operational Attention and drill-downs](./operational-attention.md) for the shared rules and Daily Brief revalidation behavior.
